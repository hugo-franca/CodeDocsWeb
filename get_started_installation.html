Installation

Follow the steps below to install Sem-Nome Flow.

1. Installing PETSc
Sem-Nome Flow requires the library PETSc to be compiled. You can install PETSc with the following steps:
  1. Go to the PETSc official page and download the latest source tarball.
  2. Extract the tarball using the command: 
    3. tar xf petsc-<version number>.tar.gz
  3. Enter the folder that was created from the extraction. 
    4. cd petsc-<version number>
  4. Run the PETSc configuration script with the command below. Note: if you are installing PETSc in a cluster system, you may need to execute this script directly inside the node where your simulations will be run in the future.
    5. ./configure --with-cc=gcc --with-cxx=0 --with-fc=0 --download-f2cblaslapack --download-mpich
  5. After the configuration (step 4) is done, the script will show you a command on the terminal to compile the code. Copy that command and run it. The command will be something similar (but maybe not exactly) to this:
    6. make PETSC_DIR=/home/hugo/petsc-3.16.1 PETSC_ARCH=arch-linux-c-debug all

2. Installing Sem-Nome Flow
To install Sem-None Flow, follow these steps:
  1. Download the source code tarball clicking [here].
  2. Extract the tarball. 
    3. tar xf CodeFlow.tar.gz
  3. Enter the folder that was created from the extraction. 
    4. cd CodeFlow
  4. Open the file called "makefile" on your text editor of choice. Modify the line below and replace that path with the path corresponding to your PETSc folder.
    5. PETSC_DIR=/home/hugo/petsc-3.16.1

3. Compiling Sem-Nome Flow
To compile Sem-None Flow, follow these steps:
  1. Enter the folder with the source code (if you're not there yet). 
    2. cd CodeFlow
  2. Compile by running the command below. 
    3. make main

4. Installing Valgrind (optional)

If you're going to make changes to the source code, it is highly recommended that you also install the debugging tool Valgrind. Valgrind will be extremely helpful at finding segmentation faults or memory access bugs that you may have introduced during your programming. Whenever you get an error, run the program with Valgrind and it will likely tell you exactly where the error is coming from.

To install Valgrind, you can use the command below. 

sudo apt-get install valgrind

5. Running a simulation

After installing and compiling the code through the steps above, you can run a simulation. To run a simulation you will need a simulation (SIM) file, which contains all the parameters and geometry to be used. Check the examples page to learn how to create a SIM file.

After you have your desired SIM file, you can run the simulation with the command below. Replace simulation_file with the name of your SIM file. 

./main -pc_type lu -arquivo simulation_file.sim
